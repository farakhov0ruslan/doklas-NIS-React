# ATLAS — Персональный планировщик путешествий

ATLAS — это веб-приложение для самостоятельного планирования городских путешествий. Пользователь не просто ищет достопримечательности — он собирает персональный маршрут: проходит короткий опрос, выбирает места, и получает готовый план поездки по дням с расписанием и стоимостью.

Проект реализован как **полностью клиентское приложение** без реальных API — все данные и «умная» логика работают через моки и локальные алгоритмы.

## Предметная область

Городской туризм — планирование короткой поездки (1–7 дней) по российскому городу. Пользователь имеет конкретные интересы (музеи, еда, музыка, шопинг и т.д.), ограниченный бюджет и время. Приложение помогает за несколько шагов перейти от «хочу куда-то поехать» к конкретному расписанию с адресами, временем посещения и стоимостью.

## Функциональность

### Онбординг и персонализация
- Пятишаговый опрос: интересы, стиль поездки, бюджет, темп, количество дней
- На основе ответов система подбирает релевантные места через mock API
- Предпочтения сохраняются в `localStorage` — при повторном визите ничего не нужно вводить заново
- Можно пропустить опрос и зайти напрямую в каталог
- В любой момент можно изменить предпочтения через футер

### Каталог мест
- Карточки сгруппированы по категориям: музеи, еда, музыка, экскурсии, шопинг, экстрим
- Фильтр по категории через таб-бар
- Расширенный фильтр: минимальный рейтинг и максимальная цена (bottom sheet)
- Сортировка: по рейтингу, цене, длительности, названию
- Поиск по названию, тегам и адресу (real-time)
- Scroll-reveal анимации карточек через `IntersectionObserver`

### Глобальный поиск (Cmd/Ctrl+K)
- Оверлей поверх любой страницы
- Навигация стрелками, выбор Enter, закрытие Escape
- «Смотреть все результаты» — переход в каталог с предустановленным запросом

### Карточка места
- Полноэкранное фото, категория, рейтинг, длительность, стоимость
- Адрес и часы работы
- Описание и список услуг (специфично для категории)
- Теги
- Кнопки «Добавить в подборку» и «В избранное»

### Корзина (подборка мест)
- Добавление/удаление мест из любой точки приложения
- Просмотр итоговой стоимости и суммарного времени
- Переход к построению маршрута

### Планировщик маршрута
- Алгоритм автоматически распределяет места по дням с учётом продолжительности
- Расписание: автоматические временны́е метки для каждого места
- Перестановка мест внутри дня
- Перенос места между днями
- Удаление места из плана

### Итоговый план
- Сводка по всем дням: места, время, стоимость
- Агрегированные метрики: число мест, суммарное время, бюджет
- Текстовые заметки к поездке
- Поделиться маршрутом: Web Share API или копирование в буфер обмена
- Экспорт в `.txt`-файл
- Печать через `window.print()`

### Избранное
- Добавление в избранное с карточки и со страницы детали
- Список избранных мест — отдельная страница
- Счётчик в шапке приложения

### История просмотров
- Автоматически сохраняет последние 10 просмотренных мест
- Отображается в каталоге в виде горизонтального скролла

### Тёмная тема
- Переключатель в шапке
- Учитывает системные настройки (`prefers-color-scheme`) при первом визите
- Сохраняется в `localStorage`

### Адаптивность и UX
- Мобильная вёрстка (от 320px)
- Плавные переходы между страницами
- Skeleton-карточки во время загрузки
- Toast-уведомления (успех, ошибка, инфо)
- Диалог подтверждения деструктивных действий
- Кнопка «вернуться наверх» при скролле
- Баннер офлайн-режима при потере сети

## Как опрос влияет на подборку мест

После завершения онбординга приложение отправляет предпочтения на `POST /api/recommendations`. Алгоритм `getRecommendations` применяет их в следующем порядке:

### 1. Интересы → жёсткий фильтр по категории

Из всех мест остаются только те, чья категория выбрана пользователем.

```
Выбрал: Музеи + Еда
→ Исключаются: Музыка, Экскурсии, Шопинг, Экстрим
```

### 2. Бюджет → жёсткий фильтр по цене

| Выбор | Лимит цены |
|---|---|
| Экономно | до 500 ₽ |
| Комфорт | до 1 500 ₽ |
| Без ограничений | любая цена |

Бесплатные места (цена = 0) проходят фильтр при любом бюджете.

### 3. Стиль поездки → мягкий приоритет внутри подборки

Стиль не убирает места, но поднимает наверх категории, наиболее ему соответствующие (через рейтинговый буст +0.5):

| Стиль | Приоритетные категории |
|---|---|
| Культура и история | Музеи, Экскурсии |
| Вечеринки и ночная жизнь | Музыка и бары |
| Природа и парки | Экскурсии, Экстрим |
| Гастрономический тур | Еда |
| Приключения | Экстрим, Экскурсии |
| Всего понемногу | без приоритетов, только по рейтингу |

### 4. Темп + Количество дней → лимит выдачи

Итоговое число мест в подборке = темп × дни:

| Темп | Мест в день | Пример: 3 дня |
|---|---|---|
| Спокойный | 4 | 12 мест |
| Умеренный | 6 | 18 мест |
| Активный | 10 | 30 мест |

Из отфильтрованного и отсортированного списка берутся первые N мест — лучшие по рейтингу с учётом стилевого буста.

### Итоговая логика

```
все места
  → [интересы] оставить только нужные категории
  → [бюджет]   убрать дороже лимита
  → [стиль]    отсортировать: приоритетные категории идут первыми
  → [темп×дни] обрезать до нужного количества
```

## Технологии

| Технология | Применение |
|---|---|
| **Vue 3** (Composition API, `<script setup>`) | UI-компоненты, реактивность, жизненный цикл |
| **TypeScript** (strict) | Типизация всех сущностей, store, composables |
| **Vite** | Сборка, HMR, dev-сервер |
| **Vue Router 4** | SPA-навигация, именованные маршруты, query-параметры |
| **Pinia** | Глобальное состояние: `userStore`, `placesStore`, `cartStore`, `plannerStore`, `favoritesStore` |
| **MSW v2** (Mock Service Worker) | Перехват fetch-запросов в браузере и в тестах, имитация REST API |
| **Vitest** + **@vue/test-utils** | Юнит-тесты store, composables, компонентов |
| **IntersectionObserver API** | Директива `v-intersection` для scroll-reveal анимаций |
| **Web Share API** / **Clipboard API** | Шаринг маршрута |
| **localStorage** | Персистентность: предпочтения, корзина, план, избранное, тема |
| **CSS Custom Properties** | Дизайн-система (цвета, типографика, отступы, тени) с поддержкой тёмной темы |

## Архитектура

```
src/
├── assets/styles/       # Глобальные стили, переменные, reset
├── components/
│   ├── common/          # AppHeader, AppFooter, BaseButton, Toast, Modal, SearchOverlay…
│   ├── onboarding/      # Шаги опроса: интересы, бюджет, темп, стиль, дни
│   ├── places/          # PlaceCard, PlaceCardCompact, CategorySection, Filters, SortBar…
│   ├── planner/         # DayBlock, PlaceInDayRow
│   └── cart/            # CartItemRow, CartSummary
├── composables/         # useCart, useFavorites, useSort, useFilters, useTheme,
│                        # useToast, useSearchOverlay, useRecentlyViewed, usePlacesInit…
├── directives/          # v-intersection (IntersectionObserver scroll-reveal)
├── mocks/               # MSW handlers: /places, /places/:id, /recommendations, /route/*
├── router/              # Маршруты приложения
├── stores/              # Pinia stores
├── types/               # Place, UserPreferences, DayPlan, Category…
├── utils/               # Алгоритм планировщика, форматтеры, константы категорий
└── views/               # HomeView, OnboardingView, PlacesView, PlaceDetailView,
                         # CartView, PlannerView, SummaryView, FavoritesView
```

### Ключевые архитектурные решения

- **Module-singleton composables** — `useTheme`, `useToast`, `useSearchOverlay`, `useRecentlyViewed` держат состояние на уровне модуля (вне компонента), что даёт глобальный синглтон без store
- **`usePlacesInit`** — composable, который гарантирует загрузку данных о местах при переходе на любую страницу, зависящую от них (Planner, Summary, Cart). Решает проблему потери данных при перезагрузке страницы
- **MSW в тестах** — один и тот же слой моков используется и в браузере, и в Vitest, что избавляет от дублирования фикстур

## Mock API

| Метод | Эндпоинт | Описание |
|---|---|---|
| `GET` | `/api/places` | Все места (40+ объектов) |
| `GET` | `/api/places/:id` | Детали конкретного места |
| `POST` | `/api/recommendations` | Подборка по предпочтениям пользователя |
| `POST` | `/api/route/generate` | Генерация маршрута из корзины |
| `POST` | `/api/route/recalculate` | Пересчёт маршрута (легче / оптимизировать / добавить) |

## User Flows

### Flow 1 — Новый пользователь, полный сценарий

```
Открывает приложение
  → Видит hero-секцию с поиском
  → Нажимает «Показать» → появляется OnboardingModal
  → Выбирает «Настроить подборку» → OnboardingView (5 шагов)
    [1] Выбирает интересы: музеи, еда
    [2] Выбирает стиль: культурный
    [3] Выбирает бюджет: средний
    [4] Выбирает темп: умеренный
    [5] Выбирает 3 дня → «Готово»
  → PlacesView: персональная подборка загружена
  → Листает карточки, открывает детали мест
  → Добавляет 7-8 мест в подборку
  → Нажимает кнопку корзины → CartView
  → «Составить маршрут» → алгоритм генерирует план
  → PlannerView: 3 дня, переставляет места внутри дней
  → «Сохранить» → SummaryView
  → Добавляет заметки, скачивает .txt, делится маршрутом
```

### Flow 2 — Пользователь пропускает онбординг

```
Открывает приложение
  → Нажимает «Показать» → OnboardingModal
  → Выбирает «Не сейчас»
  → PlacesView: все места без персонализации
  → Фильтрует по категории «Музеи»
  → Открывает панель фильтров, ставит рейтинг ≥ 4.0
  → Добавляет понравившиеся места в подборку
  → Строит маршрут
  → В футере нажимает «Настроить подборку» → проходит онбординг
  → Возвращается в каталог — подборка персонализирована
```

### Flow 3 — Возвращающийся пользователь

```
Открывает приложение (данные из localStorage)
  → Предпочтения сохранены, онбординг не показывается
  → Нажимает Cmd+K → открывается глобальный поиск
  → Вводит «Эрмитаж» → видит результат, нажимает Enter
  → PlaceDetailView: открывает детали, добавляет в избранное ❤️
  → Переходит в «Избранное» через шапку
  → Из избранного добавляет место в подборку
  → Идёт в корзину → строит маршрут
```

### Flow 4 — Редактирование маршрута

```
PlannerView: маршрут уже построен
  → Видит день 1: 4 места, день 2: 3 места
  → Нажимает «✕» рядом с местом — удаляет его
  → Стрелками меняет порядок мест в дне
  → Переносит место из дня 1 в день 2
  → «Сохранить →» → SummaryView
  → Видит обновлённую стоимость и суммарное время
  → Добавляет заметку: «взять зонт, купить билеты заранее»
  → Нажимает «🖨️ Печать»
```
